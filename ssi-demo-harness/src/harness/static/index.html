Instruction actions

Enter a count and run an action. Each button drives the instruction-service API with seeded ZITADEL users.

Create instructions

Create draft instructions with varied LOB, type, and creator.

Submit instructions

Submit up to N DRAFT instructions as middle office (mo-100).

Approve instructions

Submit (if needed) and approve up to N DRAFT/SUBMITTED instructions.

Reject instructions

Reject up to N SUBMITTED instructions with a seeded approver.

Suspend instructions

Suspend up to N APPROVED instructions.

Reactivate instructions

Reactivate up to N SUSPENDED instructions.

Run instruction policy scenario

Execute the fixed 9-step OPA policy harness (create, deny, submit, approve, list versions).

Payment actions

Create and manage payments against approved instructions. Requires at least one approved instruction.

Create payments

Create payments against approved STANDING instructions using seeded payment creators.

Update draft payments

Bump amount on DRAFT payments to the next club tier (creates a new version).

Submit payments

Submit up to N DRAFT payments for approval as the original creator.

Approve payments

Approve up to N SUBMITTED payments as pay-204 (covers all LOBs).

Reject payments

Reject up to N SUBMITTED payments as pay-204.

Run payment policy scenario

Execute the 7-step OPA payment policy harness: create (DRAFT), deny unauthorized create/submit/approve (ALERT), submit (SUBMITTED), approve (APPROVED).
